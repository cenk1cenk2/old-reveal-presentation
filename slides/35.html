Causes

  • Increase in developer count
  • At core have to use the same programming language
  • Complicated planning on more core level
  • Proving your idea to be worthy

Problems

  • Many branches, many conflicts
  • Other programming languages may have better solutions
  • Reduces the time for development, while most of it is taken by planning
  • Can reduce motivation

With the increase of the developer count as it is required in a larger project, some problems might surface with this workflow.

---PUSH---

The most trivial one being, with many people working on the same project there will be more branching out which might cause conflicts.

---PUSH---

While calling scripts with different programming langauge is possible, the core is mostly written using the language that is decided at the beginning. But other programming languages for a certain part of the application can have better libraries or even better performance in certain tasks.

---PUSH---

With many people working on the same project, there have to be a good coordination which requires meticolious planning. This planning can go very deep to be sure that there will be no conflicts. This will reduce the time for the actual development and reserve more for planning and meaningless meetings.

---PUSH---

It is usually the case when you are working with many other people, to prove your ideas to be worthy. So the developer is forced to not apply their ideas and creativity which reduces motivation as well as might lengthen the development time.
