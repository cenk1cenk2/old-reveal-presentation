A command line interface that is working with a plugin based microservice architecture.

  • Control and monitor remote servers.
  • Control and monitor Docker containers.
  • Deploy Docker containers to remote servers utilizing Ansible.
  • Manage all servers in a central location.

This proof of concept will be utilized as a base for my next open source project.

I aim to cut down on modularity and create a new version for my CLI interface where it can monitor docker as well as the server itself.

This application will consist of 3 main parts which are the CLI itself, frontend-backend and a daemon.

While CLI can be used independently it can be further expanded with the frontend-backend configuration to enable remote control over web.

Multiple daemons from multiple servers can connect through a message queue to extend this feature over many servers.
