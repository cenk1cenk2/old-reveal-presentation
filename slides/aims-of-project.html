Aims of the Thesis

To create a common interface for:

  • IOT Devices
  • Physical Computers
  • Virtualized Local Servers
  • Remote Servers

What can you do with IOT devices?

  • Control their respective functions
  • Control power states

What can you do with computers?

  • Control Power States
  • Monitor Hardware Health and Utiliziation
  • Monitor and Manage Running Services on Servers

So what do I aim for this thesis when it is completed.

I aim to create common interface for IOT devices, Physical computers, virtualized servers which might use different hypervisors and remote servers.

This might look a little ambitious considering even the brands behind them ignoring to design their own proper interfaces, but it is not intended to fully control these devices through the interface if they have interface of theirselves.

Rather it will be a complementery interface for them to control the functions that are not very advanced and I frequently need to do on the go. For more advanced tasks it can redirect to the already established interfaces of these programs.

What is to with IOT devices you might ask.

This will be the more complete side of the project controling even more advanced functions of these devices.

Because I am using an iphone with 16gbs of storage after my phone broke down and as you know its storage management, it is not possible to even install all different applications of variety brands of iot devices on it.

So this shall be a common interface that I dont have to jump around to control variety of devices.

-------PUSH------

Again you might ask why not even use the interfaces of established programs on the server side, and use everythings respective interfaces for server control.

These tools as I mentioned does not have a proper monitor interface and even if they have it is on the paid side of things.

So this will be a place to monitor all the devices, and with all the devices I mean I can monitor multiple devices at once.

Some basic functions like wakeonlan, which I mainly use for daily computers like my gaming computer or virtualized HTPC to watch movies and this stuff requires me to go through SSH to even wake them up. And with this day and age we are getting lazier to get up and press power button on a computer that is in an other room. Therefore with these basic functions like waking, sleeping, shutting down devices will be also implemented.

Automation tools also have keeping alive the services built-in. But the problem is again with the interface. Eventhough I assume the services are always alive, sometimes problems happen and I need to restart them or somewhat. I use a custom supervisor to run all the services on those servers inside their respective docker containers. With this way utilizing what I already use, I can monitor all the services that are essential to run on the startup and monitor or manage them through a interface where I can see all my services inside a common interface, and just filter out services that are not essential to run at boot time.
